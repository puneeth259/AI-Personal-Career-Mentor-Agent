`db.save(user)` is blocking the async event loop. You must `await` a dedicated async DB driver.

"Explain how Redis works as an event-driven cache but using a real-world restaurant example."

Excellent question. Let's break this down into a highly conceptual but real-world analogy.

Imagine a massive, sprawling restaurant (Your Database, e.g., PostgreSQL). It has millions of ingredients stored in deep freezers. If every customer asks for a complex dish, the chef has to run to the back, gather items, cook them, and plate them. This takes **time (latency)**.

Now, imagine placing a small, ultra-fast food warmer right next to the waiter station. This is Redis (Remote Dictionary Server). It stores pre-made, hot, ready-to-go plates (Key-Value pairs in RAM) of only the absolute most popular dishes right now.

Real-World Flow (The Architecture):

[Client Request] --> GET /user/profile
       |
       |--> 1. Hits Waiter (Application Backend)
       |
       |--> 2. Waiter checks the Warmer (Redis Cache)
       |       [Key: user_101_profile]
       |
       |------> a) FOUND (Cache Hit) -> Instantly served! (~1ms)
       |
       \------> b) NOT FOUND (Cache Miss) ->
                   |
                   \--> 3. Waiter runs to Kitchen (PostgreSQL DB)
                   |       (Slow disk read, ~50ms)
                   |
                   \--> 4. Food served AND placed in the Warmer for the next guy!
